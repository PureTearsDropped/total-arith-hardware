# ゲート回路の条件表 — (U,V,W) 双線形ユニットを 論理ゲートで 組む前の 設計契約

`c = W·( (U·a) ⊙ (V·b) )`（前線形 → R 個の並列乗算 → 後線形）を **ゲートから** 組み直す。
組む前に、これまでの全域算術で 確立した規則を **「入力→出力の契約」** として 洗い出す。
各条件は 後で ゲートが 実装すべき 振る舞い。**未確定の分岐は 【要確定】** と記す。

出所: `bfp_sed.py`（代数コア）, `sd2_core.py`, `sd2_gates.py`, `saturating_arith.py`, `SPEC.md`。

## A. 数の表現（1 本の線が 何を運ぶか）

- **A1 符号つき桁** `t ∈ {−1,0,+1}` を 2 本 `(p,n)` で: `enc(t)=(t==1, t==−1)`, `dec(p,n)=p−n`。
  冗長ゼロ `(1,1) ≡ (0,0) = 0`（同じ値の複数表現・値は厳密）。
- **A2 ブロック浮動** 成分値 = W 桁の 仮数 × `2^E`。**E は ブロックの M 成分で 共有**（配線シフトで整列）。
- **A3 予約レール** `±MAX`（上に飽和・全桁）／`±MIN`（最小の非零・最下位1桁）。
  別記号でなく、**フラグが 予約の意味を 与える** 通常のビット列。
- **A4 NaN/Inf のビット列は 存在しない**（全域化の不変条件）。

## B. 各成分に 付く 状態（フラグ）

- **B1 以上** ge（bit0, ≥）: `|真| ≥ |表示|`。
- **B2 以下** le（bit1, ≤）: `|真| ≤ |表示|`。
- **B3 組** (ge,le): `00=厳密` / `10=≥` / `01=≤` / `11=境界なし（不明）`。
- **B4 符号不明** sunk: 表示の **符号が 保証されない**（別ビット）。

## C. 全域化の規則（回路が 必ず 実装・NaN/Inf は 決して 出さない）

- **C1** `x × 0 = 0`（厳密）。［部分積が 全部 0 で 自動］
- **C2** `x ^ 0 = 1`（空積）。
- **C3** `a / 0` = **【要確定・分岐】**
  - (i) **Moore–Penrose `a/0 = 0`**（代数コア `bfp_sed`）＝ 符号を 捏造しない。**推奨（原理的ユニット）**。
  - (ii) **飽和 `a/0 = sign(a)·MAX`, `0/0 = 0`**（`saturating_arith`）＝ 大きさを 残すが 符号を 作る。
  セデニオンでは `N(b)=Σbᵢ²=0 ⟺ b=0` なので 判定は N の 零検出 1 個。
- **C4 溢れ**（E が Emax 超 / 仮数が Emax で W 桁超）→ **±MAX に飽和・ge を立てる・符号は保つ**。
- **C5 潰れ**（非零仮数が MIN より下へ 完全に落ちる）→ **±MIN・le を立てる・符号は保つ**。
- **C6 全演算が [−MAX,+MAX] に飽和**。除算器だけでなく **乗算器も** 飽和させる
  （`saturating_arith` の教訓: `…×MAX` が 次の積で 即 範囲外＝Inf。だから 全演算で 飽和）。

## D. 回路が 保つべき 区別

- **D1 二種の 0**: 真の0（全桁0・厳密な相殺 or 零因子）→ `(ge,le)=00, 仮数0`。
  潰れた0（アンダーフロー）→ `±MIN, le=1`。**混同しない**（別出力）。

  **概念（利用者）: ±MIN は「潰れた0」でなく、有限機械における 符号つき無限小 ε。**
  古典解析の「0 に限りなく近いが 0 でない・向きを持つ量（dx）」を、無限に小さくできない機械が
  **床 MIN で止めたもの**。`le`（≤）が これを 忠実にする: `±MIN・le` = 「|x| ≤ MIN・符号既知」
  ＝ **無限小の帯 (0, MIN] に 向き付き**（0⁺ か 0⁻）。特定の小さい値でなく「そちら向きの無限小」。
  - **符号捏造が消える理由そのもの**: `1/(本当の0)=0`（中立・向きなし・Moore–Penrose）／
    `1/(0⁺)=+大(→+MAX)`, `1/(0⁻)=−大`（ε は 向きを持つ ⟹ 方向極限が 正しく出る）。
    IEEE は 0 を 常に 0⁺ 扱いして `1/0=+inf` と **向きを捏造**する。**0 と ε(±) を分けた瞬間、
    古典の方向極限が 戻る。** ＝ 二種の0を 区別する 本当の御利益。
  - **有限の限界**: 機械 ε は **一段だけ**。真の無限小の塔（ε ≻ ε² ≻ …・ε²=0 の双対数）は 持てない
    （MIN の下は 床＝何もない ⟹ `MIN×MIN` は 0 か MIN に 落ちる）。
    ＝ **有限性で 一段に 切り詰められた 無限小**。
- **D2 符号の保存**: 飽和・潰れの値は 飽和前の値の 符号を 保つ。
  **例外**: 相殺する 符号つき和で 符号が 決まらない → `sunk=1`。

## E. 演算ごとの 契約

- **E1 乗算**（R セルの 各々）:
  - 厳密入力: フラグは 正規化からのみ（切り捨て→ge / 溢れ→ge / 潰れ→le）。**積は sunk を 立てない**。
  - フラグ付き入力: 出力成分ごとに 符号つき区間を 伝播。単調→ge/le、相殺→境界なし＋（区間が0跨ぎなら）sunk。
- **E2 加減**（U,V,W の 線形層）: **E を整列**（小E側を 左シフト＝無損失）→ 桁ごと 3:2 圧縮。
  同じレールで 飽和した 異符号成分が 相殺するとき **sunk を立てる**（符号不明の 唯一の発生源）。
- **E3 除算**（ユニットが 割るなら）: `a·conj(b) / N(b)`。
  スカラー除数→締まる `(≥)/(≤)→≥`。セデニオン除数→依存性で 保守的（境界なし）。
  `N=0`→C3。除数区間が 0 跨ぎ→**境界なし＋sunk、値は 0 でない**。

## F. 型・次元・指数（型違い）

- **F1 次元 M**: 両オペランドは 同じ M（配線 OMEGA が M 依存）。異なる M → **型エラー**、
  または CD 塔で 埋め込み（新しい虚部を 0 詰め）。
- **F2 捻れ**: **型 = (M, OMEGA/群, E)**。同じ M でも 捻れが違えば（行列 vs 除算代数）**意味的型不一致**。
- **F3 指数 E**: 加減は E 整列、乗算は E 加算。ブロック共有Eの 維持は 正規化の 責務。
- **F4 (U,V,W) の形**: `U:R×m, V:R×n, W:p×R`。`a:m, b:n, c:p`。`U·a` と `V·b` は 共に 長さ R
  （⊙ の 前提）。長さ不一致 → 型エラー。

## G. 零因子・±MIN の 注意（セデニオン M≥16）

- **G1 零因子** `a≠0, b≠0` で `a·b=0` が 在る → 真の0（D1）。正しく 潰れと 区別。
- **G2 ±MIN 規約の 破れ**: 予約語 ±MIN は `a·b=0 ⟹ a=0 or b=0` を 仮定するが、これは **M≥16 で破れる**。
  ⟹ 回路は「MIN×非零 ≠ 0」を **仮定してはならない**（セデニオン積は ±MIN でも 厳密0 になりうる）。
  直しは `L(x)` の 零空間（【要検討】）。

## H. 組み上げた ユニットの 振る舞い（3 層）

- **H1 前線形 `U·a, V·b`**: 符号つき整数の 線形結合（Strassen は 成分 {−1,0,+1} ⟹ **加減だけ・乗算器なし**）。
  各々 E2 に従う（整列・相殺で sunk・C4/C5 飽和）。
- **H2 R 個の 要素ごと積**: 各々 E1。
- **H3 後線形 `W·(·)`**: E2。
- **H4 フラグの流れ**: 3 層を 貫いて 伝播（前和が 溢れ ge → その積が 継承、後和が 区間規則で 合成）。

## ゲート実装の 三つの 規律（バグから 確立）

- **R1 冗長表現の罠**: 符号つき桁は 冗長 ⟹ 値の大小に 関わる 操作（溢れ判定・floor・±MIN判定）の
  **前に 必ず canonicalize**（正の非冗長形へ）。積和・付け替えは 冗長のまま で良い。
  （実例: 値1の冗長形が 高位桁を持ち 溢れ誤認／ゲート版 Newton の floor が +1 ズレ）
- **R2 方向づけ丸め**: 「打ち切り = 境界(ge/le)」は 自動でない。**単調性に 合わせて 各丸めの向きを 設計**
  して 初めて 健全（rsqrt: 内側 y² は ceil・外側 floor ⟹ 下界 0/80000。floor 一律だと 65601/100000 破れ）。
- **R3 底2の 付け替え**: 仮数の底 = 指数の底 = 2 ⟹ E の変更 = 桁位置の 再ラベル ＝ 無損失。
  ブロック正規化 = 一度解決 → 先頭位置 → 付け替え → E調整（大きさ判定は ここ だけ・核の演算には 不要）。

## FPGA 実装（openXC7 完全オープンフロー・Vivado 不使用）

**経緯**: Vivado 2026.1 は 無償枠が Windows 限定に 制度変更（Linux は 有償）→ 61GB 削除し
openXC7（yosys 0.62 + nextpnr-xilinx 0.8.2 + prjxray）を ソースビルド（タグ名 yosys-0.62→v0.62 修正・
AMD 出荷スクリプトの CRLF バグも 発見/修正）。

**面積 実測（yosys synth_xilinx・Artix-7）**:
| モジュール | LUT | A7-100T 占有 |
|---|---|---|
| sd_add2 N=16 | 256 | 0.4% |
| sd_mult10 | 3,717 | 5.9% |
| blocknorm（フラグ込） | 8,160 | 12.9% |
| blocknorm フラグ除去 | 7,488 | 11.8% |
| sed_comp（融合MAC） | 21,438 | 33.8%（**余裕で搭載可**） |
| top_arty（UART+sd_add2） | 786 | 1.2% |

**フラグのコスト（主張→測定で 訂正）**: 差分 672 LUT = **正規化ユニットの 8.2%・演算パイプ全体の ≈2.3%**
（「1%未満」は 甘かった。それでも ECC の 12.5% より 安い）。

**配置配線（nextpnr）**: top_arty 395 SLICE_LUT・**Fmax 244.86 MHz**（100MHz 制約に 2.4 倍余裕・
クリティカルパスは UART 側 = 定数深さ加算器は ボトルネックでない）。

**ビットストリーム**: fasm2frames + xc7frames2bit → **top_arty.bit（3.7MB）生成済み**。
残るは ボード到着 → openFPGALoader 書き込み → host_test.py 実機照合 のみ。
物理配線 vs 論理配線の 対応も 整理: U,V,W を LUT 焼き込み（差替=再合成）か 表駆動（差替=メモリ書込）は
設計選択（3層トレードオフの 物理版）。
