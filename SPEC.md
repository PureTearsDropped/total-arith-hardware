## この形式は、何か

**一言で: マイクロスケーリング（MXFP / ブロック浮動小数点）の、セデニオン・3値符号つき2進・状態つき版。**

```
業界の MXFP（GPU）            この形式
32 個のスカラーで指数を共有   →  **16 成分のセデニオンで指数を共有**
2 値の桁 {0,1} + 2^E         →  **3 値の桁 {−1,0,+1} + 2^E**（符号つき2進、Avizienis 1961）
指数だけ                     →  **指数 + 状態（符号不明・境界）**   ← 研究の中身
```

**底は 2 のまま**（GPU と同じ、指数変更 = ビットシフト、無料）。**桁だけ 3 値 {−1,0,+1}**。
これが研究の芯である —— 桁が 0 に対して**対称**なので:

* **符号反転 = 全桁の p↔n 入れ替え = 配線**（ゲートゼロ。対の原理）
* **符号は仮数が運ぶ**（別の符号トリットは要らない。残るのは「符号不明」1 ビットだけ）
* `gate9` が **符号の代数**（積の符号は符号の積）／`{0,1}` の非対称な桁ではこれが崩れる

**そして GPU が捨てている情報（どちらへ溢れたか・符号が分かるか）を、値に持たせる。**

## 1. 数の形式

```
テンソルの被演算子 = 2^m 個のセデニオン
  各セデニオン = **1 ブロック**（16 成分が指数を共有）
    ブロック指数 E                        （整数。**浮動する**。2^E が値のスケール）
    成分 c_0 … c_15                       （各成分 = 下記の「成分」）
    値_k = 2^E × 仮数_k
```

**これは 固定小数点ではなく ブロック浮動小数点である**（対話で判明。当初「可変固定小数点」と呼んでいた）。
**指数 `E` は固定でなく、演算に連れて浮動する**（積で足され、正規化で動く）。
**radix の粒度 = セデニオンごと**（16 成分で 1 指数）。MXFP と同じで、ブロックごとにスケールが動く。

### 1.1. シフトはどこで起きるか — 3 種、うち 2 つは無料（測定済み）

**固定小数点は「積のあと必ず rescale」でシフトを強制していた。浮動なら消える:**

```
**乗除算** : 指数を足すだけ（E_out = E_a + E_b）      → **シフトなし**
             （現行 cs_inject(acc, **p+q**) が既に指数の足し算。実測 rescale 0 回）
**加減算** : 別ブロックで E が違えば 整列が要る
             → 小さい方の仮数を |E_a−E_b| ずらす = **重み添字の付け替え = 配線（無料）**
             （※ 同一ブロック内の 16 成分は E 共有 ⟹ テンソル積の和には整列すら要らない）
**再正規化** : 結果の仮数を W 桁に収める（任意。深い網で幅を抑えるため）
             → ずらして 窓からはみ出した下位桁を捨てる = **唯一の損失 → 潰れ → ≤ フラグ**
```

> **シフトが要るのは 加減の整列（無料）と 最終正規化（そこだけ損失）だけ。**
> **積和は full-width で貯めて、正規化を最後に一度**（carry-save MAC。現行 `cs_mac` がこれ）。
> **⟹ そして その唯一の損失（正規化の潰れ）が、状態符号 `≤` の唯一の発生源である**（§11 の `要検証` がここで発火）。

### 1.2. 指数を動かす = 重み添字を変えるだけ（桁は動かさない）

**現行プログラムでは、重みは配列の添字である**（`sed/seddigit_carrysave.py` を読んで確認）:

```
cs_inject(acc, w, digit)         桁ベクトル（16 桁）を 重み w に入れる
値 = Σ_w b^w × digit[w]          位置 w = 指数（現行の底 b = **3**、設計目標 b = **2**）
桁上げ : w → w+1                 乗算 : cs_inject(acc, **p+q**, ...) ← 添字の足し算
```

**⟹ 指数を変える = 同じ桁ベクトルを、別の重み添字に入れるだけ**（桁は 1 本も動かない。**底に依存しない**）:

> **指数の演算 = 添字の演算 = 配線・アドレッシング。仮数は一切動かない。**
> ブロック指数 `E` は**メタデータ（小さい整数）**であり、変えても桁は触らない。
> 乗算で指数が足される（`E_out = E_a + E_b`）のも、**値の掛け算ではなく添字の足し算**である。
> 底が 2 なら **シフト = ÷2（= 2⁻¹）**、あなたの言う「2^-1」がこれ。

**現行の乗算器は 底3（`3^n`）である**（`from_trits` が `Σ·3^k`、`to_trits` が `divmod(·,3)`）。**2^n ではない。**
**底が入っているのは 2 箇所だけ**（実測で切り分け）:

```
**底に依存しない**（乗算器の骨格・そのまま流用）:
  gate9（桁の積 {−1,0,1}×{−1,0,1}→{−1,0,1}）  ← ただの値の積。底2でも底3でも同じ表
  重み配置 p+q（指数の足し算）
**底が入っている**（底2 に差し替える対象）:
  gate27（3:2 圧縮器 x+y+c = same + **3**·high）  → 底2 は same + **2**·high
  to_trits / from_trits（**3^k** の符号化）        → **2^k**
```

**⚠️ 監査人の誤り（訂正済み）**: 当初「3 値 ⟹ 底 3」と決めつけ、「`×3`/`÷3` のシフト」と書いた。
**値の個数（3）と 位置の底（2 か 3）は 独立である。** `{−1,0,+1}` は 底 2 でも使える（符号つき2進）。
**base-10 も base-3 も要らない。底2 でシフト=÷2、指数=2^E、GPU と底が揃う。**

**損失は、指数変更そのものからは起きない:**

```
桁を別の重みに移すだけ           → **無損失**
保持する窓（幅 W）から 下へはみ出した桁を捨てる → **潰れ ⟹ b=≤ フラグ**
```

> **「変える」= 添字（無料）。「捨てる」= 窓からはみ出したときだけ（フラグ）。** 二つは別の操作である。

**⟹ そしてこれが、状態符号が仕事を持つ場所である。** [`FINDINGS.md`](FINDINGS.md) §11 は
「セデニオン回路には出番が無い（整数で厳密だから）」と結論した。**保持窓を有限にした瞬間、
はみ出した桁を捨てる操作が `≤` フラグを立て始める。** §11 の `要検証`（rescale を深くすれば発火）が、
**この形式ではデフォルトで起きる。**

## 2. 成分の符号化

**1 成分 = 仮数（W 桁）＋ 境界トリット ＋ 符号不明ビット。**

```
**仮数**         W 個の {−1, 0, +1} 桁、位置 **2^k**（符号つき2進。(p,n) 符号化）
                 値 = Σ d_k · 2^k。**符号は仮数が運ぶ**（0 に対称なので −x が必ず在る＝対の原理）
                 ⟹ **別の符号トリットは要らない**（{−1,0,+1} 自身が符号を持つ）
                 冗長（|D|=3 > 底2）⟹ 桁上げ無しの加算（carry-save）が保てる

**境界トリット** b ∈ {−1, 0, +1} = **≤** / **=** / **≥**
                 = : 仮数がそのまま真値
                 ≤ : |真値| ≤ この大きさ（**潰れた**。窓からはみ出した下位桁を捨てた）
                 ≥ : |真値| ≥ この大きさ（**溢れた**。仮数が W 桁を超えた）
                 「境界なし」= (仮数 0, b=≥) ＝ 中身の無い下界 ← 4 番目の状態は不要

**符号不明ビット** 1 ビット。立つと 仮数の符号は無意味（|仮数| だけが有効）
                   **加減でのみ立つ**（§8。積・商では絶対に立たない ＝ 唯一の交差点）
```

**⟹ 三つが 別々に分かれている**（平衡三進はこれを 1 桁に混ぜていた）:

```
小数点（分数） → ブロック指数 E（§1.1。無料）        桁には入れない
符号（±）      → 仮数の桁 {−1,0,+1}（対の原理）       別トリット不要
符号 不明      → 符号不明ビット（加減の飽和でだけ）
大きさ         → 仮数の |値|
```

**この形式での MAX / MIN は、W と E で決まる**（float64 の定数ではない）:

```
MAX（仮数）= 2^W − 1        ← W 桁の {−1,0,+1} で表せる最大（Σ 2^k）
MIN（仮数）= 1              ← 最下位桁 2^0（この radix での分解能）
溢れ  : 仮数が MAX を超える     → 仮数 = MAX、b = ≥、**符号は保つ**
潰れ  : はみ出した非ゼロを捨てる → 仮数 = MIN、b = ≤、**符号は保つ**
```

## 4. 演算が状態をどう動かすか（[`FINDINGS.md`](FINDINGS.md) §8）

**積・商 — 符号と境界は並列に流れる（互いを見ない）:**

```
符号   : 符号の積          ← gate9（既存。0=不明 は吸収的）
境界   : (≤a)(≤b)→≤ab   (≥a)(≥b)→≥ab   **(≤a)(≥b)→境界なし**   ← gate_bmul(4) + vacuous(3)
指数   : E_out = E_a + E_b
**符号不明は 積・商では絶対に生まれない**（符号は符号の積で決まるので、大きさを失っても残る）
```

**加・減 — 符号が境界の式を選ぶ（唯一の交差点）:**

```
**同符号** → 境界 = p + q       符号は確定
**異符号** → 境界 = max(p, q)   **符号不明**（大きさの比較に失敗 ⟹ 符号ビットが立つ）
片方が確定(=v) → v−q ≤ |x| ≤ v+q（両側の境界）
```

**⟹ MAX/MIN は向きが逆なので、加減の結果も端で逆になる:**
上端（≤MIN）の引き算は締まる（`2·MIN`→`MIN`）／下端（≥MAX）の引き算は消える（境界なし）。

### 4.1. 積・商も 代表値で計算＋方向保持 → 量的な境界（利用者の改善。`interval_mul.py`）

**当初は「非厳密入力の 積/商 → 一律 境界なし」と 潰していた。潰しすぎ。** 利用者の指摘:
**「MAX/MIN を その値として 計算し、`MAX/(MAX/2)` なら `≥2` とすればいい」。**

各成分の 真値を フラグから **符号つき区間** `[lo,hi]` にし、積（符号つき和）を **区間演算**で伝播、
結果区間から フラグを引き直す（`add` の区間足し算と 同じ形を 積へ）:

```
**単調な場合**（スカラー・厳密値との 掛け割り・reciprocal）: **量的境界**
   (≥1000)/(=500) → **≥2**   (≥100)×(=3) → **≥300**   1/(≥MAX) → ≤(1/MAX)
**相殺する場合**（セデニオン積の 混ざる成分）: 区間が広がり **自動で 境界なし**（正しい）
```

**実測**: 厳密スカラーとの 掛け割りで **量的境界 63%**（以前は 全部 境界なし）／セデニオン積は 100% 境界なし。

### 4.2. 完全な区間 積・商（両オペランド・符号不明・除数0跨ぎ。`interval_adversarial.py`）

利用者の続き: **「除数区間も 乗数区間も」「除数区間が 0 を跨ぐと 符号も値も 不明（0 ではない）」
「0 と 乗除算しなきゃ 0 に ならない」。** これらを 完全な区間演算で:

- **両オペランドの フラグ + 符号不明** を `_ival` で 区間に（符号不明 ⟹ 対称区間 or 全体）
- **積**: `_ac_interval`（符号つき和を `_isum` で ∞ 安全に）→ `_flags_for`（0 跨ぎ ⟹ 符号不明）
- **商**: `q = ac / N`（`_idiv`。`N=Σb²≥0`、**`Nlo=0`＝除数が 0 になりうる ⟹ 大きさ 無限・符号不明**）。
  **代表値は `ac/N`（非零）のまま ⟹ 0 に ならない**（本当に `N=0`＝厳密な `a/0` のときだけ 0）

**敵対的健全性**（両フラグ・符号不明入力・`N` が 0 も・相殺を誘発する 揃った成分）:
**0 違反 / 384,000 チェック × 4 種**（`mul`/`div` × 両フラグ/片フラグ）。

### 4.3. 成分ごとに 量的境界も 出る／破れる規則との 区別（利用者）

**訂正（監査人の 雑な一般化）**: 「セデニオン積は 常に 境界なし」は 誤り。**成分ごとの 区間**が、
和が **相殺しない 成分**では **量的境界（≥/≤）** を、相殺しうる成分では その和の時点で **境界なし/符号不明** を
出す（実測: 疎な `≥` 入力の積で 4 成分に `≥`・境界なし 0）。**「常に境界なし」は 成分を 揃えて 相殺を
誘発した 敵対的テストの 数字を 一般化しただけ。**

**盲目的な 直接規則 `(≥)/(≤)→≥` は セデニオン成分で 9% 破れる**（商成分 = 16 個の ±積の和 ⟹
`|a|` を大きく `|b|` を小さくしても 個々の成分は 相殺・零因子で 小さくなりうる。実測 9%）。
**⟹ 直接規則は 使わず、区間で 和を 見る**（相殺なら 境界なし・嘘をつかない）。

### 4.4. 除数が スカラーなら 締まる／セデニオンなら 依存性で 保守的

**利用者の規則 `(≥a)/(≤b)→≥(a/b)` は スカラーで 健全**（1 成分 = 大きさ ⟹ 0 違反）。
**除数が スカラー（実数 b₀）なら `a/b = a_k/b₀` の 成分ごと ⟹ 依存性なし ⟹ 締まる**:

```
(≥1000)/(=500) → ≥2     (≥1000)/(≤500) → **≥2**（前は境界なし・今 締まる）
(≤1000)/(≥500) → ≤2     (≥1000)/(≥500) → 境界なし     除数が 0 跨ぎ → 符号も値も 不明
```

**除数が セデニオン**のときは `a·conj(b)/N(b)` で `b` が 分子分母 両方に 出る **依存性問題**（区間演算の
既知の限界）⟹ `(≥)/(≤)` が 数学的に `≥2` でも **境界なし**に 緩む。**健全（嘘はつかない）・締まりだけ 緩い。**

### 4.5. affine/中心化形式で 締めようとした 結果 = 実質 利得ゼロ（測定済）

依存性を 締めるべく **中心化形式（平均値形式）**を 実装（`_centered_div_intervals`, `USE_CENTERED`）:
平均値の定理で `q=num/N` を `q ⊆ q̄ + [∇q](箱)·(箱−中心)` と 囲む（偏微分の商則が num と N の 相殺を 織り込む）。
**健全（全敵対テスト 0 違反）だが 除算の 境界なし回収率は 有界ランダムで 0.00%・近スカラーで 0.2%。**

**効かない 数学的理由**（＝実装の失敗でなく 構造）:
1. **支配（大きさ）成分は 符号が確定なら naive 区間が 既に締めている**（例 `[0.624,0.625]→≥`）。
2. **残る 境界なし成分は 真に 0 を跨ぐ**（LE フラグが 0 を含む・セデニオンの 符号つき和の 相殺は 本物）
   ⟹ どんな 相関追跡でも 符号は 付けられない。
3. **GE/NB は 真に 非有界** ⟹ 境界なしが 正しく、中心化は 発動しない（有界箱が 前提）。
4. **除算では `b` が 分子に 線形・分母に 二乗** ⟹ `N` 区間は 有界で 既にタイト ⟹ 依存性の ずれは
   二次で 小さい（`x/x` の 一次相殺とは 違う）。**だから naive で 十分。**

⟹ `USE_CENTERED=False` を 既定に（健全なコードは 残すが 利得ゼロ・診断は `div_tighten_regimes.py`,
`div_centered_test.py`）。**セデニオン除算の 境界なしは 本物の 情報欠如で、締める余地は ほぼ無い**が 結論。

## 5. テンソル積の中での伝播

現行の `sed/sedenion_tensor_logic.py` は **配線 XOR（k = i^j）＋ 符号 OMEGA[i,j] の入れ替え**。状態を乗せる:

```
出力成分 k = ⊕(入力 i,j : i^j = k) の 積項の和
  各積項の符号 = OMEGA[i,j] × 符号(x_i) × 符号(y_j)   ← gate9 で合成（乗算器ゼロ）
  各積項の境界 = 境界(x_i) の gate_bmul 境界(y_j)      ← 並列
  **和（桁加算器の木）で 符号不明が生まれうる**        ← §4 の加算規則
指数 : E_out = E_x + E_y、和のあと 各段で再正規化
```

**費用**: 状態の伝播は 1 成分あたり新規 7 ゲート（`gate_bmul` 4 + `vacuous` 3）。
セデニオン積 1 個の桁ゲートは 数千個 ⟹ **0.1% 未満**。

**⚠️ 底を 3→2 にした影響**: `gate9`（符号の積）と状態のゲートは**そのまま生きる**。
**作り直すのは 桁の圧縮器だけ** —— 現行の `gate27` は平衡三進の 3:2 圧縮器。
底 2 の符号つき桁 `{−1,0,+1}` では、**符号つき2進の加算器**（carry-free。Avizienis の桁加算）に置き換わる。

### 5.1. 零因子は、この回路では問題にならないかもしれない（**要検証・重要**）

[`FINDINGS.md`](FINDINGS.md) §10: `±MIN` 規約は `a·b=0 ⟹ a=0 or b=0` を仮定し、**セデニオンの零因子で破れる**
（`(e1+e10)(e4−e15) = 0` を「潰れ」と誤診）。**だがこの回路では解けるはずである:**

```
現行の設計は 和を 厳密に計算してから 一度だけ rescale する（static_tree + ripple）。
⟹ 零因子の和は **rescale の前に厳密に 0**  → b = **=**（確定した 0）
⟹ 潰れの 0 は **rescale が桁を落とした**     → b = **≤**（|x| ≤ MIN）
⟹ **境界トリットが、二つの 0 を見分ける。** L(x) の零空間を計算する必要が無い。
```

**これは 実装で確認した**（`bfp_sed.py` 自己テスト③）: `(e1+e10)(e4−e15)` は 全成分 境界 `=`（構造的 0）、
潰れた 0 は 境界 `≤`。**境界トリットが 掛ける前の値を見ずに 二つの 0 を見分ける。**
**⟹ §10 の限界は この回路の構造（厳密和 → 単一正規化）が回避する。** `L(x)` の零空間は要らない。

## 10. 任意の 2^k 次元への 一般化（`nd_algebra.py`）

**利用者**: 「これを n 次元の代数と することは できますか？」

**できる。ほぼ無料**（`nd_algebra.py` で 実/複素/四元/八元/セデニオン/32/64 を 実測）:

| M | 名前 | 可換 | 結合 | 零因子 | 底2積==ref |
|---|---|---|---|---|---|
| 1 | 実数 | ✓ | ✓ | 無 | ✓ |
| 2 | 複素 | ✓ | ✓ | 無 | ✓ |
| 4 | 四元数 | **×** | ✓ | 無 | ✓ |
| 8 | 八元数 | × | **×** | 無 | ✓ |
| 16 | セデニオン | × | × | **有** | ✓ |
| 32, 64 | | × | × | 有 | ✓ |

**次元に依存するのは 符号表 `OMEGA` だけ**（Cayley–Dickson で 再帰的に 自動生成、`e_i·e_j = ±e_{i⊕j}` は
全 `2^k` で 成立）。**桁の積・18 ゲート圧縮器・ブロック浮動・状態（符号不明/境界）は 全て 成分ごと**
⟹ `M` を変えるだけで 任意次元。

> **Cayley–Dickson の 性質喪失が そのまま 現れる**（可換 ≤2 → 非可換 4 → 非結合 8 → 零因子 16+）。
> **八元数(8)まで 零因子なし ⟹ `±MIN` 規約が 綺麗／セデニオン(16)以上 零因子あり ⟹ 境界トリットが 処理。**
> **この設計は 零因子の在る次元(16+)でこそ 真価が要る**（16 未満は 普通の除算代数で済む）。

**限界**: 真に任意の `n`（2 の冪でない）は `XOR routing` を 持たない ⟹ 一般の構造定数テンソルが要り、
**乗算器フリー（符号=配線・経路=XOR）を 失う**。**`2^k` の塔が この設計の 自然な一般化。**

### 10.1. 符号表（twist）を変えると 行列積になる（`matrix_algebra.py`）

**利用者**: 「M を変えたら 色んな行列積が できると思う」「実数なら 実行列の 同時要素数を 増やせる？」

**その通り**（実測。q=1,2,3 で 結合的・行列積一致・sd2 積一致 すべて ✓）:

| twist | 得られる代数 |
|---|---|
| **Cayley–Dickson** | 実/複素/四元/八元/セデニオン（除算代数の塔・§10） |
| **symplectic（Pauli/clock-shift）** | **`M=2^{2q}` → `2^q × 2^q` 実行列積**（M=4→2×2, 16→4×4, 64→8×8, 256→16×16） |

**経路は 全て 同じ XOR（`k=i⊕j`）、変わるのは 符号表 OMEGA だけ ⟹ 同じ乗算器。**
基底 `e_g`（`g∈(Z/2)^{2q}`）を パウリ `⊗_i X^{a_i}Z^{b_i}` に対応 ⟹ twisted 積 = 行列積。

> **M を上げると 実行列の 同時要素数が 増える**（M 成分 = √M×√M 実行列）。
> **効率**: √M×√M 行列積の 実仕事は `M^1.5` 積和だが 群代数の積は `M²` 項 ⟹ **√M 倍 冗長**。
> ⟹ **GPU テンソルコアと同じ「小タイルとして使い、大行列は タイルで並べる」**（符号=配線の分 冗長は 多少埋まる）。

**⟹ 一つの XOR 配線ハードが: 代数の塔（Cayley–Dickson twist）も 行列タイル（symplectic twist）も、
どちらも NaN を出さず・状態が正直に・任意次元で。twist を差し替えるだけ。**
