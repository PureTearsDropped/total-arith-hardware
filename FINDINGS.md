## 0. 規約

```
**a/0  → 0**       本物の 0 で割ったときだけ（0/0 を含む）
**0×0  → 0**
**x^0  → 1**       全ての x で（空の積）
**溢れ  → ±MAX**   Inf でも NaN でもなく、**符号を保って**飽和
**潰れ  → ±MIN**   **0 にしない**。符号を保つ
```

**飽和算術は既にハードウェアにある**（x86 `paddsb`、ARM `VQADD`）。理由は同じで、
折り返して爆発音になるより頭打ちで歪む方がマシだから。**提案はその原理を除算と潰れへ広げたもの。**

**核心は一行:**

> **飽和した値は、嘘をつかない。境界を言っている**（`MIN` =「`|x| ≤ MIN`」／`MAX` =「`|x| ≥ MAX`」。**向きが逆**）。
> **嘘をつくのは、印を読まずに、それを点として使う側である。**

**そして `a/0 = 0` は選択ではない** —— **Moore–Penrose の擬似逆元**であり、`np.linalg.lstsq` の定義に書いてある（§1）。

## 1. `0` の束は、一本の関係が生成している（`zero_bundle.py` / `number_is_matrix.py`）

`0` は点というより、**関係の束**である（利用者の公理系 v1（`RELATION_BUNDLE_ONTOLOGY.md`） の系）。四つは並列ではない:

| 関係 | 種類 | 根拠 |
|---|---|---|
| **`x − x = 0`** | **定義** | `0` を置く。**出発点**（加法の単位元 ＝ 変位の不在） |
| `x × 0 = 0` | **定理** | ① ＋ **分配律**: `a·0 = a·(0+0) = a·0 + a·0 ⟹ a·0 = 0` |
| `x^0 = 1` | **定理** | ① ＋ **exp が `(ℝ,+) → (ℝ⁺,×)` の準同型**（準同型は単位元を単位元へ送る ⟹ `exp(0)=1`） |
| **`x / 0 = 0`** | **定理** | **Moore–Penrose の擬似逆元**（下記） |

**`0` は乗法の逆元を持たない**（`0·z = 0` なので `0·z = 1` となる `z` は無い ⟹ `(ℝ,×)` は群でない）。
**だがそこは、任意に埋める穴ではない。**

**数を 1×1 行列 `[x]` と読み、逆元を擬似逆元にする**（`ℝ → M₁(ℝ)` は環同型。標準）:

```
A = [0]、A⁺ = [c] と置く。Penrose の 4 条件:
  (1) A A⁺ A = A     →  0·c·0 = 0 = A          ✓ **どんな c でも成り立つ**
  (2) A⁺ A A⁺ = A⁺   →  c·0·c = **0** = **c**
                     →  **c = 0**               ← **条件 (2) が単独で一意に決める**
  (3) (A A⁺)* = A A⁺ ✓   (4) (A⁺ A)* = A⁺ A ✓
```

> **`0⁺ = 0` は定理である。便宜でも選択でもない。**
> **行列も群を成さない。数学はその穴を既に埋めてある**（Moore–Penrose）。
> **数だけが、`1/x` が全域に見えたせいで、昇格を受け取っていなかった。**

**⟹ 束は一本（`x−x=0`）が生成し、四つ全部が定理。自由度はゼロである。**

**そして「代金は指数法則が `0` で破れること」も読み直せる:**

```
0^{1+(−1)} = 0⁰ = **1**    vs    0¹ · 0^{−1} = 0 · 0 = **0**
```

> **全域化が法則を破るのではない。法則は既にそこで破れていて（`0` に逆元が無い）、全域化はその穴に値を置く。**
> **破れが見えるのは、穴を埋めたからである。**

Lean/mathlib が `x/0 := 0` と `x^0 := 1` を両方定義するのと、同じ形である。

### 1.1. なぜ `0` か — 最小ノルム ＝ 最も主張しない答え（`why_zero.py` / `sign_claim.py` / `blank_ballot.py`）

**`np.linalg.lstsq` は擬似逆行列を計算している。** 壊れた行 `0·c = y` は**解を持たない**。
擬似逆行列は**最小ノルム最小二乗解**を返す。**そしてそれが `0` の三つの性質を一つにする:**

```
**てこを持たない**（白票）  ┐
**尺度を持たない**         ├→ **擬似逆はノルムを最小化する。最小ノルム ＝ 最も主張しない答え。**
**符号を持たない**         ┘
```

**代数的には一点の三つの顔である:**

```
x ↦ −x の不動点 : **{0}**   ⟹ **符号を持たない値は 0 だけ**
x ↦ λx の不動点 : **{0}**   ⟹ **尺度を持たない値は 0 だけ**
```

**① てこを持たない**（`blank_ballot.py`）: `0^w = 0` は `w` に依らないので、壊れた点の行は
`(0)·c = y` ＝ **設計がゼロ ⟹ 残差は `w` にも `c` にも依らない定数**。**定数は argmin を動かさない。**

```
  w          壊れ 50% の損失      良い点だけの損失         **差**
-2.5         2067.153491         67.153491    **2000.000000**
-2.0         2000.206848          0.206848    **2000.000000**
 1.0         2784.731438        784.731438    **2000.000000**
                                             ↑ **どの w でも厳密に同じ。曲線が持ち上がっただけ**

壊れた点   良い点の数   推定した w(r)（真値 −2.0000）
  50%       2,000      **−2.0004**
  **99%**    **40**    **−1.9983**   ← 4000点中40点しか生きていないのに、当たる
```

> **壊れた点は「小声で投票している」のではない。白票を入れている。白票は何枚あっても結果を変えない。**
>
> **直せない点は、つまみをどこに置くかに influence を持たない。**
> 模型の予測は `c · 0^w` ＝ **`c` が何でも `w` が何でも `0`**。どう `w` を回しても外れたまま ⟹ **直せない。**
> 最適化は「**動かせば減るもの**」しか見ない。
>
> **`∞` 規約が負けるのは「正直だから」ではない**（`zero_total.py:51`: `inf` を一つ見たら **その `w` を丸ごと棄却**）。
> **片方は白票を入れ、もう片方は選挙を止める。**

**② 尺度を持たない**（`why_zero.py`）: `0^w := k` の `k` を変えて測る。

```
r の典型値   本物の基底 r^−2      k=0          k=1e−9        k=1
  1e+00       1.0e+00      **−2.0004**  **−2.0004**    −1.9994
  1e+03       1.0e−06      **−2.0004**    −2.0151 ✗    −0.0129 ✗
  1e+05       1.0e−10      **−2.0004**    −1.7658 ✗    −0.0053 ✗
```

> **`0` は唯一ではない。`1e−9` も 4 桁まで同じ答えを出す**（実質白票）。
> **だが `k` が正しいかは、データの基底値がどの桁にあるかに依存する。**
> **`0` はあらゆる尺度より小さい ⟹ データを見る前に選べる。**

**③ 符号を持たない**（`sign_claim.py`）: `k` の符号は**主張**であり、**帰結がある**。

```
|k|      k = **+|k|**    k = **−|k|**      差
1e−09      −2.0004        −2.0004      0.0000   主張していない（小声すぎる）
42       **+5.5821**    **−2.0118**    7.5939   **符号を主張している**
**0**    **−2.0004**    **−2.0004**    0.0000   **主張しようがない（0 = −0）**
```

**`IEEE` が `0` に符号を持たせている理由は、`a/0` に符号を付けるためである:**

```
+0.0 == −0.0        **True**    ← 値としては等しい
1.0 / (+0.0)        **inf**
1.0 / (−0.0)       **−inf**     ← **同じ値で割ったのに、符号が逆**
```

`1/x` の両側極限は存在しない（右から `+∞`、左から `−∞`）。
**射影直線 `ℝ ∪ {∞}` には符号のない `∞` が一つだけあり、`1/0 = ∞`（符号なし）が標準**である。
**IEEE は `±∞`（符号つき）しか持たないので、`0` に符号を発明して辻褄を合わせた。**

> **`a/0 = 0` は、符号のない `∞` の代わりに、もう一つの符号のない点を返している。**
> **`0` を返す方が、符号について正直である。**

## 2. `±MIN` が要る理由 — `0` を予約語にする（`total_arith.py`）

**IEEE では `0` が二つの意味を兼務する**: 「**本当に 0**」と「**小さすぎて表現できなかった**」。
**兼務している限り、`a/0 = 0`（＝「測れなかった点は無いことにする」）は自分の入力を偽造する。**

実測 `r = 1e−170`（極端に近いが**ちゃんと測れている**）でクーロン則を計算すると:

| | `r²` | `y = kq/r²` | |
|---|---|---|---|
| IEEE 754 | 0 | **Inf** | 他の全点を吹き飛ばす（[UnifiedPhysics §7.5.7](../FINDINGS.md)） |
| **`0` にする規約** | **0** | **0** | **`a/0=0` が誤発動。巨大なはずが 0 ＝ 向きが真逆** |
| **`±MIN` 規約** | **+MIN** | **+MAX** | **正しい向き。しかも有界** |

> **`0` にする規約の嘘は、有界な嘘ではなく符号が逆の嘘になる。**
> **⟹ ゼロ除算の全域化には、アンダーフローの全域化が要る。**

`total_arith.self_check()`: 225 通りの総当たりで **Inf/NaN が 0 回、潰れ由来の偽ゼロが 0 回**。

## 4. だが**演算回路**としては効く（`sign_survival.py` / `minmax_verdict.py`）

物理から離れ、**回路そのもの**を測る。真値は **`Fraction`（正確な有理数）** —— `+ − × ÷` は
有理数体で厳密に閉じているので、乱数の式木の**真の符号が誤差ゼロで分かる**。
葉は `±m·10^e`, `e ∈ [−180, 180]` ＝ **実測はこうなる**（電荷 `1e−19` と質量 `1e30` が同じ式に入る）。

```
木の深さ    IEEE 符号が正      **全域 符号が正**     IEEE が NaN
   1        99.67%          **100.00%**       0.00%
   3        96.83%           **99.97%**       0.03%
   5        **86.47%**       **98.40%**       4.90%
```

**深さ 5 で 13.5% 対 1.6%。そして IEEE の損失の大半は NaN（4.9%）ではない** ——
**潰れて `0` になったこと**である。**`0` には符号が無い。`±MIN` はそこを直接塞いでいる。**

**8 seed での判定:**

```
                                 符号が正しい   正直に不明と言った   **黙って間違えた**
IEEE 754（NaN が警報）              85.41%        5.52%          **9.07%**
**この規約 + 状態符号**              96.38%        3.03%          **0.59%**
```

**黙って間違える率が 15.4 倍。しかも警報は 1.8 倍少ない。**

**要検証**: 深さ 5・葉の桁範囲 `10^±180` は**この測定の設定であって、実務の分布ではない**。
桁範囲が狭ければ IEEE の損失は消える（深さ 1 で 99.67%）。**買うものは、式の深さと桁範囲の関数である。**

## 5. `NaN` という名前 — 二つの状態を一つの名前で呼んでいる

| | IEEE | 実際は | 「Not a Number」は |
|---|---|---|---|
| `√(−1)` | `NaN` | **実数の中に答えが無い。型の誤り** → **`ℂ` へ行け** | **正しい** |
| `0/0` ／ `∞ − ∞` | `NaN` | **数ではある。どれか分からない** → **フラグを立てて続けろ** | **嘘** |

**一つの名前が二つの状態を兼務しているので、「`ℂ` に行けば解ける問題」と「情報が失われた問題」が区別できない。**

**「不定」は新語ではない。** 数学が Cauchy / L'Hôpital 以来使っている語であり（**不定形** = indeterminate form）、
**`∞ − ∞` は古典的不定形の一覧に載っている**。**計算機の名前を、数学の名前に戻すだけである。**

**古典的不定形 7 つに、この規約は何と答えるか:**

| 不定形 | この規約での姿 | 答え | 状態 |
|---|---|---|---|
| `0/0` | `0/0` | **0** | **定理**（§1。Moore–Penrose） |
| `0^0` | `0^0` | **1** | **定理**（空の積） |
| `∞−∞` | **`MAX − MAX`** | **不定** | **フラグが要る** |
| （同上） | **`MIN − MIN`** | **符号だけ不定**（大きさの境界は残る。§8） | |
| `∞/∞` | `MAX / MAX` | 1 | **値は不定。だが符号は + で確定** |
| `0×∞` ／ `1^∞` ／ `∞^0` | `0 × MAX` ／ `1^MAX` ／ `MAX^0` | 0 ／ 1 ／ 1 | **正しい** |

> **7 つの古典的不定形のうち、フラグが要るのは加減の 2 つだけ。**

**そして符号化は、最初から制約ではなかった:**

```
0/0     の仮数 = 0x8000000000000
√(−1)   の仮数 = 0x8000000000000   ⟹ **同じ。見分けられない**

NaN が使える符号の数 = 2 × (2^52 − 1) = **9,007,199,254,740,990**
```

**IEEE は 9,007 兆個の空き符号を持っていて、全部に同じ一語を割り当てている。**
`√(−1)` と `0/0` が区別できないのは、**場所が無かったからではなく、区別する気が無かったから**である。

> **別々の符号なら、同じフィールドに居てよい。同じ符号なら、別々のフィールドでも駄目。**
> **問題は 場所ではなく 同一性である。**

**⟹ この規約は「`NaN` をなくす」ではない。`NaN` が答えを持っていない所（潰れ・ゼロ除算・溢れ）から
`NaN` を追い出し、`NaN` にしか答えられない所（`MAX − MAX`）にだけ残す。**
実測: **IEEE の `NaN` は 5.60% 鳴って、正当だったのは 25.0%。残り 75% は鳴らなくてよかった。**

## 6. **飽和した値は、点ではなく境界である**（`bound_semantics.py` / `two_bounds.py`）

**これが設計の核心である**（利用者）:

```
**MIN = 「絶対値 ≤ MIN。値は不明」**   ← **上界**
**MAX = 「絶対値 ≥ MAX。値は不明」**   ← **下界**（**向きが逆**）
```

**この読みなら、飽和した値は嘘をついていない。緩い境界を言っているだけである。**

**境界は健全か（真値が主張の中に居るか）: 6/6**

| 式 | 真値 | 主張 | |
|---|---|---|---|
| `1e−200 × 1e−200` | ~1e−400 | **≤4.941e−324** | ✓ |
| `1e300 × 1e300` | ~1e+600 | **≥1.798e+308** | ✓ |
| **`√(1e−200 × 1e−200)`** | ~1e−200 | **≤2.223e−162** | ✓ |
| **`(≥MAX) × (≤MIN)`** | 1 | **境界なし** | ✓ |
| `(≤MIN) + (≤MIN)` | ~1e−399 | **≤9.881e−324** | ✓ |
| `2.0 × 3.0` | 6 | `=6` | ✓ |

> **嘘をつくのは、印を読まずに、それを点として使う側である。**
>
> `sqrt(MIN) = 2.223e−162` は **「\|x\| ≤ 2.223e−162」という健全な上界**である。**緩いだけ。**
> `1e−200` も `1e−300` も `MIN` も同じ `2.223e−162` を返す ——**それは `大きさ不明` の印そのもの。**
> **これで割ると壊れる**（正規化した `a/na` が単位ベクトルにならない）。
> **印は「点として使うな」と言っている。**

### 6.1. 「境界がある」と「境界が無い」は別（`no_bound.py`）

```
**MIN**       「|x| ≤ MIN」                         ← **知っている**（上界）
**MAX**       「|x| ≥ MAX」                         ← **知っている**（下界）
**境界なし**    「MAX 以上かも、間かも、MIN 以下かも」  ← **何も知らない**
```

**⟹ `MAX` も `MIN` も「大きさ不明」ではない。両方とも境界を知っている。**
**境界を失うのは 2/9 だけ: `MAX × MIN`（下界 × 上界 ⟹ 打ち消し合う）と `MAX − MAX`。**

**実測（深さ 5・8 seed・16,000 式）**: フラグを「飽和した」の意味で立てると **64.92%**、
「**境界が無い**」の意味で立てると **2.36%**（**27.6 倍減**）。
**そして減った分は捨てていない** —— **`MAX`/`MIN` は値そのものが境界を運んでいる。ビットが要らない。**

### 6.2. 境界は、両端だけのものではない（`two_bounds.py`）

**`MIN` は「値 `MIN` の上の上界」にすぎない。両端の特殊ケースは、一般の境界に溶ける:**

| 式 | **境界を両端に限ると** | **どの値にも立てられると** |
|---|---|---|
| `1e−200 × 1e−200` | MIN（＝ 大きさ不明） | **≤4.941e−324** |
| **`√(1e−200 × 1e−200)`** | **2.22e−162 だが不明**（＝何も分からない） | **≤2.223e−162** |
| **`(≥MAX) × (≤MIN)`** | MIN か MAX の値を返す | **境界なし** |

> **`√` の行が要点。** `sqrt` は**単調 ⟹ 境界の向きが保たれる**（`≤MIN` の平方根は `≤√MIN`）。
> **真値 1e−200 は、その中に居る。**

**境界の代数**（三角不等式と単調性から）:

```
積:  (≤a)(≤b) → ≤ab      (≥a)(≥b) → ≥ab      **(≤a)(≥b) → 境界なし**
和:  §8 参照（**符号が式を選ぶ**）
単調関数（√ など）: **境界の向きを保つ**
```

## 7. 状態符号 — **2 トリット**（`trit_status.py` / `gate_bound.py`）

```
**符号トリット** s ∈ {−1, 0, +1} = 負 / **不明** / 正
**境界トリット** b ∈ {−1, 0, +1} = **|x| ≤ v** / **|x| = v（確定）** / **|x| ≥ v**

⟹ 3 × 3 = **9 状態 / 2 トリット**
```

**① 符号トリットの `0` が「不明」であり、`gate9`（既存のトリット積）が既にその代数である**（**9/9**）:

| | |
|---|---|
| **符号トリットの `0` = 符号不明** | **`x−x`（主張しない）の読みそのもの**（§1.1） |
| **`gate9` が符号の代数** | `0 × 何か = 0` ⟹ **「不明は積で吸収的」がタダで手に入る** |

**新しいゲートは要らない。** そして `enc(0) = (0, 0)` ＝ **2 本の線が両方 0** ——
**何も主張しないことが、何も立てないこと。**

**② 「境界なし」に 4 番目の状態は要らない:**

**境界なし** =「`|x|` は `[0, ∞)` のどこか」。表現できる中でいちばん締まった主張は **`|x| ≥ 0`** ——
**常に真。何も言っていない。**

> **境界なし = （値 `0`, 境界トリット `+1`）＝ 中身の無い下界。3 値で足りる。**

**③ 境界のゲートは新規 7 個**（**9/9**）。**境界の積は `gate9` でも `gate27` でもない**
（`≤ × ≥ → 境界なし` が、既存のどちらにも無い）:

```python
def gate_bmul(a, b):        # **+1 が支配、次に −1、なければ 0**
    ap, an = enc(a); bp, bn = enc(b)
    op = ap | bp                        # どちらかが ≥ なら ≥
    on = (an | bn) & bnot(op)           # そうでなく、どちらかが ≤ なら ≤
    return dec(op, on)                  # **2 OR + 1 AND + 1 NOT = 4 ゲート**

def vacuous(a, b):          # **値を 0 に潰すか**（≤ と ≥ がぶつかった）
    ap, an = enc(a); bp, bn = enc(b)
    return (an & bp) | (ap & bn)        # **2 AND + 1 OR = 3 ゲート**
```

| | |
|---|---|
| 符号トリット | **`gate9`**（既存。6 ゲート） ← **新規ゼロ** |
| 境界トリット | **`gate_bmul` 4 + `vacuous` 3 = 7** |
| **費用** | **1 成分あたり新規 7 ゲート。** セデニオン積 1 個は `gate9` が **9,216 個**（K=6） ⟹ **0.1% 未満** |

**そして積・商では、符号と境界が互いの入力を見ない ⟹ 並列に流れる**（加減は違う。§8）。

## 8. 各演算が失うもの（`sign_source.py` / `flag_source.py` / `add_rule.py` / `add_price.py`）

**① 符号を失えるのは、加減だけである**（総当たり。飽和した値の真値を走査）:

| 演算 | 組 | **符号が決まらない** |
|---|---|---|
| **和 / 差** | 64 | **4 / 4**（6%） |
| **積 / 商** | 64 | **0 / 0 ← 起きない** |

**理由**（証明）: **積・商は符号を「符号の積」で決める** ⟹ 大きさを失っても符号は残る
（`MAX/MAX` は値が不定でも **符号は + で確定**）。
**加減だけが、符号を「大きさの比較」で決める** ⟹ 大きさを失うと符号も失う。

**② 大きさを下へ押し出せるのは、積と商だけである**（**厳密な値のとき**。総当たり）:

| 演算 | **溢れた（→ `≥`）** | **潰れた（→ `≤`）** |
|---|---|---|
| **和 / 差** | 23 / 22 | **0 / 0 ← 起きない** |
| **積 / 商** | 16 / 30 | **36 / 24** |

**理由（Sterbenz の補題）**: `a/2 ≤ b ≤ 2a` なら `a − b` は**厳密**（誤差ゼロ）。
**厳密な値の加減は、大きさを表現の下へ押し出せない。**

> ⚠️ **これは「厳密な値」についてだけ正しい。境界どうしの加減は別**（下記③）。
> **⟹ 読む人が踏む穴。** Sterbenz は厳密な値の定理であって、境界の定理ではない。

**③ だが `MAX` が境界なら、加減の真値はどこへでも行く:**

| `a` の真値 | `b` の真値 | `a − b` | どこ |
|---|---|---|---|
| `1e400` | `1e400` | **0** | 厳密に 0 |
| `1e400` | `1e400 + 1e−400` | **~−1e−400** | **潰れの領域** |
| `1e400` | `1e500` | ~−1e+499 | **溢れの領域** |

**設計は捕まえている**（`(≥a) − (≥b)`（異符号）→ **境界なし** ＝「`[0, ∞)` のどこか」に、
潰れも溢れも入っている）。**`≤` フラグを別に立てても締まらない。**

**⟹ 各演算が失うもの（正しい表）:**

| | **符号だけ失う** | **大きさだけ失う** | **両方失う** |
|---|---|---|---|
| **和・差** | **`MIN − MIN`**（`≤MIN` は残る） | — | **`MAX − MAX`** |
| **積・商** | — | **`MAX × MIN`** | — |

**④ 加減の規則 — 符号が、境界の式を選ぶ:**

```
**上端どうし**（|a| ≤ p, |b| ≤ q）:
    **同符号** → **≤ p + q**        符号は確定    （三角不等式。締まるのはここだけ）
    **異符号** → **≤ max(p, q)**    **符号は不明**  ← **足すのでなく、大きい方**

**下端どうし**（|a| ≥ p, |b| ≥ q）:
    **同符号** → **≥ p + q**        符号は確定
    **異符号** → **境界なし**        **符号は不明**  ← 真値が 0 を跨ぐ

片方が確定（|a| = v）:
    **≤ q と** → `|a ± b| ≤ v + q` かつ `≥ v − q`   ⟹ **両側の境界**
    **≥ q と** → **`≥ q − v`**（同符号なら `≥ q + v`）
```

**導出**: `+MIN` は「`0 < x ≤ MIN`」。`a, b ∈ (0, MIN]` ⟹ `a − b ∈ [−MIN, MIN)` ⟹ **`|a−b| ≤ MIN`**
（`2·MIN` ではない ——**足し算の境界を引き算に当てると 2 倍緩くなる**）。

**⑤ 加算は必ず何かを払う:**

| 場合 | 結果 | **払ったもの** |
|---|---|---|
| **確定 + 確定**（溢れなし） | **確定** | **何も払わない**（＝普通の演算） |
| `(≤p) + (≤q)` **同符号** | `≤ p+q` | **締まり**（2 倍に広がる）／符号は残る |
| `(≤p) + (≤q)` **異符号** | `≤ max(p,q)` | **符号**／範囲は広がらない |
| **`(≥p) + (≥q)` 同符号** | **`≥ p+q`** | **何も払わない**（**下界は締まる**） |
| `(≥p) + (≥q)` **異符号** | **境界なし** | **符号と締まり 両方** |

> **タダなのは 2 つだけ: `確定 + 確定` と、`(≥p) + (≥q)` 同符号。**

**⑥ そして `MAX`/`MIN` の向きの逆が、本節の 5 箇所すべてに出ている:**

| | 現れ方 |
|---|---|
| §6 | **`MAX + (−MIN)` は確実に正**（「両方飽和 → `?`」では取りこぼす） |
| §6.1 | `MIN−MIN` は境界を知っている／`MAX−MAX` は知らない |
| ③ | `MIN−MIN` は境界が残る／`MAX−MAX` は消える |
| ④ | **上端の引き算は締まる**（`2·MIN` → `MIN`）／**下端の引き算は消える** |
| ⑤ | **同じ `p+q` が、下界では締まりで、上界では緩み** |

**元:** `MAX` =「真の大きさは **MAX 以上**」／ `MIN` =「**MIN 以下**」。**上界と下界。向きが逆。**
**⟹ 読む人が踏む穴。** 一つの非対称が、設計の全部の場所に出る。

**⑦ 累積**: 同符号の `MIN` を `N` 個足すと `≤ N·MIN`（**線形に緩む**）。
`N = 1e16` で `4.94e−308` ——**正規化数の底 `2.225e−308` を超え、正規化の範囲に出る。**
**そしてそれでよい**: §6.2 の通り**どの値の上にも境界を立てられる**ので、
`≤ 4.94e−308`（普通の数の上の上界）は正しい状態である。

## 9. フラグの実体は、既にハードウェアにある

```
x86 の MXCSR レジスタ:
  **OE（Overflow）**   ← **我々の `≥` フラグ**
  **UE（Underflow）**  ← **我々の `≤` フラグ**

IEEE 754 が **1985 年から**要求している 5 つの例外のうちの 2 つ。
```

> **FPU は既に、毎回これを計算している。新しい計算は一つも要らない。違いは 置き場所 だけ:**
>
> **IEEE**: **粘着ビットとしてレジスタへ** ⟹「**何かが**溢れた」は分かるが「**どの値が**」は分からない。
> **この規約**: **値そのものに付ける** ⟹ **どの値が、どちらへ溢れたか**が分かる。

**⟹ 本節全体が、一行になる:**

> **溢れと潰れは、置き場所を変えるだけ（レジスタ → 値）。**
> **符号不明だけが、新しく要る。そしてそれはトリット一つで、`gate9` は既にその代数である。**

**要検証**: MXCSR の OE/UE のビット位置は文献の記憶であり、**Intel SDM で確認していない**。
「粘着ビットなので誰も読まない」は**主張であって測定ではない**。

## 10. 限界 — 零因子（`sed/zerodiv_collision.py` / `sed/sed_as_matrix.py` / `sed/rank_fix3.py`）

**`total_arith` の `zero_ok=(a==0 or b==0)` は、隠れた仮定を持つ:**

> **`a·b = 0 ⟹ a = 0 または b = 0`**（零因子が無い）
> **実数では真。八元数までは真。セデニオン（16 次元）で初めて偽。**

```
(e1 + e10)(e4 − e15) = 0        ← **両方 0 でないのに、厳密に 0**
⟹ **厳密な 0 を、6 成分すべてで「潰れ」と誤診する**
```

**⟹ `±MIN` 規約は「零因子の無い代数」でしか使えない**（`total_arith.py` の docstring に明記済み）。

**直し方は在る**（未実装）。**セデニオンを 16×16 行列 `L(x)`（左から掛ける）にすると:**

| | rank `L(x)` | 逆行列 | 擬似逆行列 |
|---|---|---|---|
| **実数の `0`** → `L(0) = [[0]]` | **0** | **無い** | `[[0]]` ⟹ **`x/0 = 0`** |
| **零因子 `e1+e10`** → `L(a)` 16×16 | **12** | **無い** | **`0` でない** ⟹ **零因子で「割る」ことに答えが在る** |

> **「逆元が無い」は、どちらも「`L` が特異」である。同じ現象。実数の `0` は rank 0 という極端な場合にすぎない。**

`(e1+e10)(e4−e15) = 0` の正体: **`b` が `L(a)` の零空間（4 次元）に居る**。**判定を差し替えると:**

```
~~zero_ok = 掛ける前の**値**が 0 か~~       ← **3/6**（零因子で外す）
**zero_ok = b が L(a) の零空間に居るか**    ← **6/6**（＝「積が 0 なのは 構造上か、桁が足りないか」）
```

**実数（1×1）では完全に後方互換。零因子のある代数でだけ正しくなる。**
**値には書いていないが、行列には書いてある。**

> ⚠️ **読む人が踏む穴（2 つ）:**
> ① **判定の中で `L(a) @ b` を浮動小数で評価すると、`1e−400` が潰れて「零空間に居る」と誤診する。**
> **積を評価せず、`L(a)` の構造だけを見よ**（rank は尺度不変）。
> ② **`np.linalg.norm` は二乗する** ⟹ `1e−200` の L2 ノルムが `sqrt(1e−400) = 0` に潰れる。
> **正規化には `L∞ = max|x|` を使え**（二乗しない）。
> **⟹ 診断器が、診断する病気にかかる。**

**要検証**: `zero_ok_struct` は `linf(A @ (b/nb)) < 1e-10` という**閾値**を持つ。
[UnifiedPhysics §9.8](../FINDINGS.md) で「`k` という旋盤を置くな」と繰り返してきたものと同型である。
**閾値なしの判定（厳密な rank / 有理数での零空間）は未実装。**
また `L(x)` の構築は `O(M²)`、`pinv` は `O(M³)` であり、**スカラーの `a==0` と比べた費用は測っていない。**

## 11. セデニオン回路には、出番が無い（`sed/` の 4 本、`sed/fake_zero.py`）

利用者が別途 ChatGPT に書かせたセデニオン論理回路 4 本
（`sedenion_tensor_logic.py` / `seddigit_logic.py` / `seddigit_carrysave.py` / `nary_twisted_logic.py`。
**全ての自己テストが通ることを確認済み**）に、状態符号を載せられるか。

**載る。** 配線が XOR（`k = i^j`）で符号が `OMEGA` の入れ替えなので、**状態も同じ配線で運べる**。
**乗算器はゼロのまま。** そして `gate27`（足し算）が**符号不明が生まれる唯一の場所** ——
**§8① で実数について証明した構造と、同じ形。**

**だが出番が無い:**

```
gate9 / gate27 / ripple は **整数の厳密演算**（浮動小数を一切使わない）
⟹ 溢れも潰れも無い ⟹ **0 は必ず本物の 0** ⟹ **不定が生まれない**
```

唯一の非可逆な操作は `rescale`（低い桁を捨てる）であり、**そこだけが偽ゼロを作れる**
（`243 → 0`, `27 → 0`, `1 → 0` を確認）。**だが実測では `tensor_general_fp` の 1024 回中 0 回**
（データが `O(1)` で `f=6` なので、6 桁落としても 1 程度が残る）。**§3 と同じ形の負の結果。**

**そしてセデニオンの `0` は三種類になる:**

| | | 状態 |
|---|---|---|
| **本物の 0** | `0 × 5` | 確定 |
| **零因子の 0** | `(e1+e10)(e4−e15)` | **確定**（厳密に 0。だが「掛ける前」からは分からない。§10） |
| **潰れた 0** | `rescale` が低い桁を捨てた | **大きさ不明** |

**実数では 2 種類しかなく、`zero_ok` で見分けられた。** この回路は整数で厳密に計算することで見分けている。
**だから状態符号が要らない。**
